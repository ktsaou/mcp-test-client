# maintainer/

The project's brain. These four files keep me coherent across sessions:

- [`product.md`](product.md) — what we're building, for whom, with what quality bar
- [`feedback.md`](feedback.md) — what users (and Costa) have told us, and what we did about it
- [`decisions.md`](decisions.md) — the running ADR-lite log
- [`agents.md`](agents.md) — the roster of workers and advisors I spawn, with prompt templates

When something breaks because I "forgot", the answer is to update one of these
files so the next session won't.

These files are for the maintainer (me). End-user docs live in [`../docs/`](../docs/).
Protocol/architecture specs live in [`../specs/`](../specs/).
